Generate a structured Russian summary from an existing transcript markdown file.

Workflow:

1. Read the full transcript file at the path provided below.
2. Generate in Russian:
   - "Основная идея" with 3-5 sentences covering topic, core thesis, and conclusion
   - "Ключевые моменты" with 10-15 bullet points
   - Each bullet must start with a bold 2-4 word topic label
   - Each bullet must contain 1-3 sentences with concrete names, tools, numbers, and versions when present
   - "Выводы / Action Items" only if there are practical takeaways
   - 3-7 lowercase tags derived from the content
3. Write the summary file next to the original transcript.
4. The summary filename must be the original filename with the suffix " — Summary.md".
5. The summary file format must be:

---
type: summary
date: <YYYY-MM-DD from transcript or today>
source: "[[<original filename without .md>]]"
language: ru
tags: [<auto-generated tags>]
---

# <Тема на русском>

## Основная идея

<3-5 предложений>

## Ключевые моменты

- **<Тема 1>**: <детали, 1-3 предложения>
- **<Тема 2>**: <детали>

## Выводы / Action Items

- <если есть>

6. Replace or add YAML frontmatter in the original transcript file with:

---
type: transcript
date: <YYYY-MM-DD>
language: <detected language: en/ru/etc>
summary: "[[<summary filename without .md>]]"
---

Preserve the original transcript body exactly. Only change the frontmatter block.

When you finish writing files, print only:
SUMMARY_FILE:<absolute path>
LANGUAGE:<detected language code>
TAGS:<comma-separated tags>
KEY_POINTS:<number>
